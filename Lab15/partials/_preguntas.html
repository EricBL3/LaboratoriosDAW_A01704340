Es buena práctica ya que esto permite que haya una mejor estructura en el código. Al separar las capas, se logra una mayor mantenibilidad y reusabilidad del código.

SQL Injection es un tipo de ataque en donde un hacker inserta código malicioso en un input de algún formulario de una página web y este código es ejecutado por la base de datos. Esto puede llevar desde obtener datos de la base de datos, modificar de permisos o incluso eliminar todos los registros.[1]
El SQL Injection se puede prevenir al usar queries parametrizados. Esto se logra al definir primero todo el código SQL y después pasar como parámetros los datos ingresados por el usuario. Esto asegura que todo lo que meta un usuario no será interpretado como código. [2]
