Preguntas

    Sin AJAX:
  1. El usuario entra a la página y manda peticiones HTTP tipo POST o GET
  2. El servidor guarda temporalmente los datos en variables de sesión
  3. El servidor regresa una respuesta HTTP mostrando los resultados de la petición
    4. Con AJAX:
  4. El usuario realiza un evento que manda peticiones XML HTTP tipo POST o GET
  5. El servidor procesa los scripts de la petición y genera el HTML necesario
  6. El servidor regresa una respuesta XML HTTP actualizando unicamente una sección específica de la página
  7. Este proceso se repite cada vez que el usuario realiza una acción que genera un evento en la página web (Siempre y cuando la funcionalidad para ello esté programada)

  • JavaScript: Hoy en día la mayoría de las funciones que tiene jQuery también se pueden programar usando JavaScript plano. Sin embargo, esto puede causar que el código sea muy difícil de entender y poco mantenible
  • Cash: Este es un plugin muy pequeño (aproximadamente 10% del tamaño de jQuery) que soporta navegadores arriba de IE10+. Su sintaxis es muy similar a jQuery aunque no ofrece absolutamente todos las características que tiene jQuery.
  • UmbrellaJS: Una libreria muy influenciada por jQuery que tiene algunas características extra comparado a jQuery. Tiene soporte para navegadores arriba de IE11+. Una característica que tiene es que puedes usar metodos nativos de JavaScript en los selectores de UmbrellaJS

Una promesa es un objeto que permite hacer de una manera más fácil peticiones asíncronas. Para lograr esto, la promesa tiene 3 estados:
  • pending: La promesa se encuentra en este estado cuando la petición sigue ejecutándose
  • fulfilled: La promesa se encuentra en este estado cuando la petición fue completada exitosamente. Al llegar a este estado, se ejecuta la función pasada en el .then()
  • rejected: La promesa se encuentra en este estado si la petición no se pudo completar Al llegar a este estado, se ejecuta la función pasada en el .catch()

  • async: Le permite saber al motor del navegador web que una función va a tener llamadas a funciones asíncronas.
  • await: Le permite saber al motor que una función es asíncrona y que debe de esperar a que se ejecute antes de continuar.
Es importante tomar en cuenta que las funciones con async regresan el resultado en una promesa por lo que es necesario usar .then() para acceder a este. Además, para manejar los errores se debe de poner todo el código en un bloque try/catch.
