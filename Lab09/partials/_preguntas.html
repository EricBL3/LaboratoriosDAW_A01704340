La función phpinfo() despliega información sobre la configuración de PHP del servidor en el que está corriendo.[1]
3 datos que llaman mi atención son:
  • Permite mostrar toda la información o pasar como parámetro un módulo de información.
  • Puede usarse para debuggear ya que contiene todos los datos EGPS (Environment, GET, POST, Cookie y Server).
  • Muestra información del equipo desde donde se está mandando a llamar la página php.

Para tener un servidor apto para un ambiente de producción se tendría que cambiar la configuración de seguridad para permitir el acceso remoto de manera segura. Además, se debe de apagar la configuración de mostrar errores y mandarlos todos a un log que será útil para debuggear y hacer cambios de último momento. [2]

El código de un archivo html se ejecuta del lado del servidor debido a que al abrir el archivo en un navegador (cliente), este manda una solicitud HTTP que es procesado por el servidor. Al terminar de procesar la solicitud, el servidor regresa todo el html que es interpretado por el cliente para desplegarlo en la pantalla. [3]
