Descripciones

URL

Un URL incorpora el nombre del dominio junto con información detallada para completar una dirección a una página web específica en un navegador. [1]

MIME

Multipurpose Internet Mail Extensions por sus siglas en inglés es un estándar que indica la naturaleza y el formato de un documento, archivo o conjunto de bytes. [2]

Métodos y códigos de errores HTTP

Indican si se completo satisfactoriamente una solicitud HTTP específica.[3] Se dividen en 5 categorías generales que son:

  1. Respuestas Informativas

    • 100 Continue
    • 101 Continue Protocol
    • 102 Processing

  2. Respuestas Satisfactorias

    • 200 OK
    • 204 No Content
    • 206 Partial Content

  3. Redirecciones

    • 301 Moved Permanently
    • 302 Found
    • 304 Not Modified

  4. Errores de Cliente

    • 400 Bad Request
    • 403 Forbidden
    • 404 Not Found

  5. Errores de Servidor

    • 500 Internal Server Error
    • 502 Bad Gateway
    • 511 Network Authentication Required

HTML5

HTML5 es en realidad un término que define a varias tecnologías incluyendo HTML. Estas tecnologías incluyen varios APIs y estándares tales como WebGL y SVG.[4]

JS

JavaScript (JS) es un lenguaje ligero e interpretado con funciones de primera clase basado en prototipos. Es utilizado para páginas web y otros entornos tales como Node.js y Adobe Acrobat. [5]

php

PHP es un lenguaje de scripting especialmente diseñado para desarrollo web. Lo que lo distingue de JS es que el código está siendo ejecutado en el servidor.[6]

jsp

Java Server Pages (JSP) es una tecnología de programación del lado del servidor que permite un método dinámico e independiente de plataformas para desarrollar aplicaciones web. [7]

Ajax

Asynchronous JavaScript and XML (Ajax) es una técnica de desarrollo web utilizado en la programación del lado del cliente para permitir enviar y recibir información de una base de datos o servidor.[8]

Preguntas

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    El internet es una la infraestructura de una red global mientras que la World Wide Web es un servicio que provee una colección de información que se accesa a través del internet.[9]

  2. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?[10]

    GET: Utilizado para consultar información del servidor. No soporta el envío del payload.
    HEAD: Utilizado para obtener información sobre un recurso determinado sin regresar el registro.
    POST: Utilizado para solicitar la creación de un registro. Soporta el envío del payload.
    PUT: Utilizado para actualizar por completo un registro existente.
    PATCH: Utilizado para actualizar un fragmento del registro.
    DELETE: Utilizado para eliminar un registro existente.

  3. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    Se debe utilizar el método POST ya que este permite el envío de payloads. Lo que significa que los datos del formulario solo van a llegar al destino al que deben de llegar y no van a poder ser vistos por cualquiera.

  4. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    Se utiliza el método GET.[3]

  5. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    Es el código OK que significa que la solicitud tuvo éxito. El éxito tiene significados diferentes dependiendo de la solicitud HTTP.[3]

  6. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    Si ya que este error significa que no se pudo encontrar el contenido solicitado. [3]

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Si ya que este error significa que el servidor encontró una situación que no sabe manejar. [3]

  8. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Un atributo depreciado/desaprobado es aquel que ya está desactualizado y puede volverse obsoleto en versiones futuras de HTML.[11] Algunos ejemplos son:
    <center>
    <font>
    <strike>

  9. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5? [12]

    HTML5 tiene soporte nativo de video y audio
    HTML5 tiene un manejo de errores consistente
    HTML5 soporta más características para aplicaciones web
    Se mejoró el soporte a dispositivos móbiles

  10. ¿Qué componentes de estructura y estilo tiene una tabla?

    Estructura:[13]
    • <caption> : titulo de la tabla
    • < table> : contenedor para datos tabulados
    • <tbody> : define un grupo de filas de la tabla
    • <td> : celda de una tabla. Debe ser hijo de <tr >
    • <tfoot> : define un grupo de filas de tablas al final de la tabla
    • <th> : encabezado de la tabla
    • <thead> : define un grupo de filas de tablas al principio de la tabla
    • <tr> : fila de la tabla
    Estilo:[14]
    • table layout
    • width
    • border-collapse
    • border
    • padding
    • text-align

  11. ¿Cuáles son los principales controles de una forma HTML5? [15]

    • <input> : define control por input dependiendo del tipo
    • <textarea> : define un área de texto para input
    • <select> : define una lista tipo drop-down
    • <button> : define un botón
    • <datalist> : define una lista de opciones pre-definidas para un input
    • <output> : representa el resultado de un cálculo

  12. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas? [16]

    Yo utilizo la versión 80 de Google Chrome en Linux y tiene una puntuación de 476/555 puntos en el test de soporte.

  13. ¿Cuál es el ciclo de vida de los sistemas de información?

    El ciclo de vida de un sistema de información ocurre en las siguientes etapas [17]:

    2. Análisis

    El ciclo de desarrollo de un sistema de información depende de que metodología se vaya a seguir.
    Algunas de las metodologías más comunes son: Agile y Lean.
